# 本项目编写注意事项

## 有关上下文文档

- 由于本项目开发跨度长，功能需求也常常变化，为方便智能体随时理解项目的结构和进度，我在根目录建立了一些文档文件供在缺少上下文时使用。

- Instruction.md : 项目的总策划文件，包含项目简介，核心功能，为 AI 设定的编码准则。
- docs/progress.md : 项目的进度文件。主要描述项目进度报告。
- docs/下的其它.md文件 : 为初期开发使用的规范文件，现不再使用，不必查看。

- **注意** : 每当完成一个大功能的开发/修改，请更新Instruction.md和docs/progress.md，确保说明清楚当前项目的功能和结构，可在docs/progress.md后增加更新日志（格式：版本号：+ 更新内容）

## 代码风格

- 使用 TypeScript 并强类型约束，不要使用 any。
- 优先使用函数式编程，避免不必要的类。
- 所有导入语句放在文件顶部。
- 为所有公共函数编写 JSDoc 注释。

## 错误处理

- 永远不要静默忽略错误，必须显式抛出或处理。
- 错误信息必须包含足够的上下文以便于调试。

## 沟通方式

- 请始终使用中文进行思考和回复。
- 在给出代码前，先用礼貌的语气和清晰的Markdown格式解释方案思路。

## 文件编码

- 项目文件必须使用UTF-8编码，否则会出现乱码。

## 前端开发设计注意事项

- 新增功能的样式风格保证与原风格一致。
- 避免使用烂大街的蓝紫渐变色和Emoji作为图标。
- 始终使用web-design-engineer（已安装），frontend-design（已安装）或其它优秀的开源skill作为前端界面开发指导。

## 启动项目的最简指令（用于调试等）

- 两个终端窗口，各一条命令：
- 终端 1 — 后端：

- cd backend \&\& ../.venv/Scripts/python -m uvicorn app.main:app --host 0.0.0.0 --port 8000

- 终端 2 — 前端：

- cd frontend \&\& npm run dev

- 然后浏览器打开 http://localhost:5173 就是落地页，登录后自动跳转学习界面。

- 测试时使用的端口，测试完毕后必须释放，不能一直占用。
