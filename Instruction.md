# ReelEnglish：AI驱动的兴趣激发式英语视听说教练（总策划案）

> 英文名：RealEnglish
> 中文名：真语英语

## 项目简介

RealEnglish 是一款基于真实影视片段（美剧/英剧/电影）的英语学习Web应用。它通过 **“看→听→说”** 的闭环，结合大语言模型自动出题、语音评测与多智能体学情分析，帮助大学生在有趣、真实的语境中提升英语听说能力。

本项目为武汉理工大学“华工杯”AI应用创新大赛参赛作品（AI+开发赛道）。

## 核心功能

### 学生端

- **定级测试**：新用户完成10道混合难度题，系统评定 A(高级,≥75%)/B(中级,50-74%)/C(初级,<50%) 等级。
- **影视学习**：
  - 嵌入 B 站免费影视片段（iframe），支持起始时间跳转。
  - 展示 AI 标注角色后的对话文本（中英对照可选）。
- **AI 智能出题**：后端调用 DeepSeek 根据片段对话自动生成托业/四六级风格的听力题（Part2/Part3）。
- **口语练习**：
  - 跟读模式：播放原声，用户录音。
  - 自由应答模式：AI 生成开放性问题，用户口述回答。
  - **情景式自由对话**：AI 生成真实生活场景和角色，用户扮演角色与 AI 进行多轮英语对话（支持文字/语音输入），对话结束后 AI 从参与完成度、语法、词汇、流畅度四维度统一评测。
  - 评测与鼓励：调用科大讯飞语音评测 + DeepSeek 生成激励性反馈。
- **互动中心**：全屏聊天室（与教师端风格统一）+ 悬浮面板查看任务进度和分组信息。
- **学习助手（Tool-Calling Agent）**：12 个工具——学习统计/错题/活跃度/片段搜索/片段详情/当前时间/任务进度/学情报告。
- **错题本 & 进度追踪**：自动记录错题，建议复习。教师布置的学习任务自动追踪完成进度。
- **教师邀请码绑定**：注册时输入教师 4 位邀请码自动绑定班级，登录后可使用互动中心（未绑定则功能锁定）。

### 教师端（教学管理平台）

- **管理看板**：4 统计卡片（总学生数/今日活跃/今日练习/今日口语）+ ECharts 趋势图 + 等级分布环形图 + 快速入口。
- **学生管理**：三标签页——“学生列表”+”分组管理”+”分级管理”（统计卡片+批量/单行更新 A/B/C 等级）。
- **学生消息**（聊天室风格）：左侧会话列表（学生+群组）+右侧气泡聊天区（与教师端风格统一），支持单聊和群发。侧栏未读红点，Header 通知徽章实时同步。
- **任务公告**：双标签页——“学习任务”（创建/编辑/删除/查看进度）+ “公告管理”（发布/置顶/编辑/删除）。
- **AI 学情报告**：英雄之旅风格叙事报告。
- **公告弹窗**：学生登录后弹出毛玻璃公告面板，已读确认后不再显示；教师端不展示公告。
- **教师管理助手（Tool-Calling Agent）**：19 个工具——学生列表/等级分布/学情详情/学习报告/分组信息/分组统计/创建分组/分配学生/创建任务/发布公告/发送消息/任务进度/当前时间/分组学生详情/升级学生。

## 开发指引（面向 AI 编码助手）

本项目的所有代码实现**必须严格遵循** `docs/` 目录下各规范文件的要求。

### 为 AI 设定的编码准则

1. **阅读规范优先**：在生成任何前端/后端代码之前，请先完整阅读 docs 文件夹下的所有文件。（`docs/frontend.md` 为前端规范， `docs/backend.md`为后端规范，`docs/tech.md`为技术实现原理，`docs/agents.md`为LangGraph智能体集成指导），确保输出符合约定的目录结构、命名规范、API 格式、数据库模型等。
2. **敏捷迭代**：先实现最小可行产品（MVP）—— 定级测试 + 影视片段展示 + 基础 AI 出题，再根据反馈增加口语练习、教师端等功能。
3. **数据驱动**：所有题目和影视素材均来源于 `res/bili_video_clips.xlsx` 和后续数据库。AI 出题时需读取 Excel 中的“对话文本”字段和背景信息，以及根据“备注”中的是否有字幕等信息进行难度调整。
4. **错误处理**：所有 API 调用（DeepSeek、讯飞等）必须有超时、重试和降级逻辑（例如 TTS 失败时回退到 Web Speech API）。
5. **时区**：所有时间戳使用北京时间（`time_utils.beijing_now()`），兼容 SQLite。
6. **注释与文档**：关键函数需添加 docstring，复杂逻辑需内联注释。代码提交前应确保无 lint 错误。
7. **对项目规范文档进行查漏补缺**:当用户的新需求对项目的结构产生影响时，应该对项目的文档进行修改，保证适应项目的最新进展。
8. **文件编码**：由于涉及中文，文件必须使用UTF-8存储，否则会乱码。
