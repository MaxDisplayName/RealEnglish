# RealEnglish 服务器部署指令

> 将此文件发给服务器 Agent 执行，执行前替换其中的 API Key 占位符。

---

## 1. 解压项目

将 `realenglish-deploy.tar.gz` 上传到服务器后：

```bash
cd /opt
mkdir -p realenglish
tar -xzf realenglish-deploy.tar.gz -C realenglish
ls /opt/realenglish/
# 应看到: backend/  frontend/  docs/  CLAUDE.md  Instruction.md  部署教程.md  项目说明文档.md  test_accounts.txt
```

## 2. 安装系统依赖

```bash
# Ubuntu
sudo apt update && sudo apt install -y python3 python3-venv python3-pip nodejs npm nginx ffmpeg

# 验证
python3 --version && node --version && ffmpeg -version
```

## 3. 后端配置

```bash
cd /opt/realenglish/backend

# 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 安装 Python 依赖
pip install -r requirements.txt

# ⚠️ 请修改下面的 API Key 为真实值
cat > .env << 'EOF'
DATABASE_URL=sqlite:///./reelenglish.db
SECRET_KEY=reelenglish-production-2026
ACCESS_TOKEN_EXPIRE_DAYS=7

# DeepSeek API（必填，否则 AI 出题/翻译/Agent 不可用）
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
DEEPSEEK_API_BASE=https://api.deepseek.com/v1

# 科大讯飞（可选，不填则使用降级模式）
XF_APPID=
XF_API_KEY=
XF_API_SECRET=

# LangSmith 监控（可选）
LANGSMITH_TRACING=false
LANGSMITH_API_KEY=
LANGSMITH_PROJECT=RealEnglish
EOF

# 初始化数据库
python init_users.py
```

## 4. 前端构建

```bash
cd /opt/realenglish/frontend

# 若网络超时先换镜像：npm config set registry https://registry.npmmirror.com
npm install

cat > .env.production << 'EOF'
VITE_API_BASE_URL=/api/v1
EOF

npm run build
```

## 5. systemd 后端自启动

```bash
sudo tee /etc/systemd/system/reelenglish-backend.service << 'EOF'
[Unit]
Description=RealEnglish Backend
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/realenglish/backend
Environment=PATH=/opt/realenglish/backend/.venv/bin
ExecStart=/opt/realenglish/backend/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable reelenglish-backend
sudo systemctl start reelenglish-backend
sudo systemctl status reelenglish-backend
```

## 6. Nginx 配置

```bash
sudo tee /etc/nginx/sites-available/realenglish << 'EOF'
server {
    listen 80;
    server_name _;
    charset utf-8;

    root /opt/realenglish/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;
    }

    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1000;
}
EOF

sudo ln -sf /etc/nginx/sites-available/realenglish /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t && sudo systemctl reload nginx
```

## 7. HTTPS 配置（麦克风必需）

浏览器要求录音功能必须在安全上下文（HTTPS 或 localhost）中使用。

### 方式一：自签名证书（快速测试）

```bash
# 生成自签名证书
sudo mkdir -p /etc/nginx/ssl
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/realenglish.key \
  -out /etc/nginx/ssl/realenglish.crt \
  -subj "/CN=服务器IP"

# 修改 Nginx 配置（在 server 块中添加）
sudo sed -i '/listen 80;/a \    listen 443 ssl;\n    ssl_certificate /etc/nginx/ssl/realenglish.crt;\n    ssl_certificate_key /etc/nginx/ssl/realenglish.key;' \
  /etc/nginx/sites-available/realenglish

# HTTP 自动跳转 HTTPS
sudo sed -i '/server_name _;/a \    return 301 https://$host$request_uri;' \
  /etc/nginx/sites-available/realenglish

# 注意：上面的 sed 会同时影响 HTTP 和 HTTPS server 块，建议手动编辑
```

更保险的方式是直接重写 Nginx 配置：

```bash
sudo tee /etc/nginx/sites-available/realenglish << 'EOF'
# HTTP → HTTPS 跳转
server {
    listen 80;
    server_name _;
    return 301 https://$host$request_uri;
}

# HTTPS
server {
    listen 443 ssl;
    server_name _;
    charset utf-8;

    ssl_certificate /etc/nginx/ssl/realenglish.crt;
    ssl_certificate_key /etc/nginx/ssl/realenglish.key;

    root /opt/realenglish/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;
    }

    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1000;
}
EOF

# 也需放行 443 端口
sudo ufw allow 443/tcp 2>/dev/null || true

sudo nginx -t && sudo systemctl reload nginx
```

> 自签名证书浏览器会提示"不安全"，点击"高级 → 继续访问"即可。正式使用建议用 Let's Encrypt。

### 方式二：Let's Encrypt 免费证书（需域名）

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 阿里云安全组放行

在 ECS 控制台 → 安全组 → 添加 443 端口入方向规则。

## 备注

- SQLite 数据库位于 `/opt/realenglish/backend/reelenglish.db`
- 如需重建数据库：`rm /opt/realenglish/backend/reelenglish.db && cd /opt/realenglish/backend && source .venv/bin/activate && python init_users.py`
- 更新部署：重新上传 tar.gz → 解压覆盖 → `systemctl restart reelenglish-backend`
